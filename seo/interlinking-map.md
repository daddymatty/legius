# Стратегія внутрішньої перелінковки — LEGIUS (legius.ua)

## 1. Модель «хаб і спиці» (hub-and-spoke)

Сайт організовано навколо **4 опорних сторінок-пілларів** (тематичні хаби), кожен з яких збирає навколо себе кластер статей (спиці) та пов'язану сервісну/локальну сторінку. Піллар отримує посилальну вагу з усіх своїх спиць і, у свою чергу, передає авторитет на комерційні сторінки.

```
                         Головна (/)
                             │
        ┌────────────┬───────┴───────┬───────────────┐
        ▼            ▼               ▼               ▼
   PILLAR 1      PILLAR 2        PILLAR 3        PILLAR 4
 simeyne-       viyskove-       korporatyvne-   podatkove-
  pravo          pravo            pravo           pravo
   │  │            │ │              │ │              │ │
 спиці спиці    спиці спиці      спиці спиці      спиці спиці
 (blog статті кластера)  ...      ...              ...
   │                                                  │
   ▼                                                  ▼
 Сервісна практика                          Сервісна практика
 /practices/family-law/                     /practices/tax-law/
   │
   ▼
 Локальні посадкові (simeynyy-advokat-kyiv, advokat-poznyaky …)
```

### 4 піллари та їх кластери

| Піллар (хаб) | URL піллара | Сервісна сторінка | Кластери-спиці (блог) |
|--------------|-------------|-------------------|------------------------|
| Сімейне право | `/blog/simeyne-pravo/` | `/practices/family-law/` | розлучення, аліменти, поділ майна, батьківські права, шлюбний договір, опіка |

## 2. Як практичні сторінки лінкують на піллари, кластери та локальні сторінки

**Сервісна сторінка практики (`/practices/family-law/`):**
- посилання вгорі/у вступі → на блог-піллар `/blog/simeyne-pravo/` (анкор: тематичний, «детальний гайд із сімейного права»);
- блок «Корисні статті» → 4–6 ключових спиць кластера (анкори = назви статей / ключі);
- блок «Послуга у вашому районі» → локальні сторінки (`simeynyy-advokat-kyiv`, `yuryst-livyy-bereh-kyiv`);
- CTA-посилання → `/contacts/` та `/team/` (профіль профільного адвоката для E-E-A-T).

**Блог-піллар (`/blog/simeyne-pravo/`):**
- посилання на всі спиці кластера (це його основна функція — розподіл ваги);
- 1 контекстне посилання на сервісну сторінку `/practices/family-law/` (комерційний анкор);
- хлібні крихти: Головна → Блог → Сімейне право.

**Спиця/стаття блогу (`/blog/alimenty-na-dytynu/`):**
- 1 висхідне посилання на свій піллар (анкор: «Сімейне право» / назва піллара);
- 2–3 горизонтальні посилання на суміжні статті кластера (анкори = ключі суміжних тем);
- 1 комерційне посилання на сервісну сторінку або локальну посадкову (анкор-CTA: «сімейний адвокат у Києві»);
- посилання на `/cases/` з релевантним кейсом, якщо є.

**Локальна сторінка (`/simeynyy-advokat-kyiv/`, `/advokat-poznyaky/`):**
- посилання на профільні `/practices/*`;
- посилання вгору на загальногеографічні сторінки (`advokat-poznyaky` → `yuryst-livyy-bereh-kyiv` → `advokat-kyiv` → `/`);
- посилання на `/contacts/`, `/team/`, `/cases/` (локальні кейси, де можливо).

## 3. Розподіл посилальної ваги з головної (link equity)

Головна `/` — найсильніша сторінка за зовнішніми посиланнями; вона передає вагу пріоритетним напрямкам через головне меню + контентні блоки:

| Пріоритет | Куди веде головна | Тип посилання |
|-----------|-------------------|----------------|
| 1 (макс.) | 4 піллари / сервісні практики (family, military, corporate, tax) | меню + герой-блок |
| 2 | Високопріоритетні локальні: `advokat-kyiv`, `yuryst-kyiv`, `simeynyy-advokat-kyiv`, `viyskovyy-advokat-kyiv` | блок «Послуги в Києві» |
| 3 | Решта практик (`/practices/*`) | меню «Практики» (мегаменю) |
| 4 | `/cases/`, `/about/`, `/team/` | меню + футер (траст-блок) |
| 5 | Блог `/blog/` (загальний хаб) + останні статті | блок «Аналітика» |
| Футер (на всіх стор.) | `/contacts/`, NAP, `/about/`, ключові практики, локальні | site-wide |

Принцип: не «зливати» вагу в маловажливі сторінки з головної. Глибокі статті отримують вагу не напряму з `/`, а через піллари (каскад). Кількість вихідних посилань з головної тримаємо керованою (≈40–60), щоб концентрувати equity на пріоритетах.

## 4. Структура хлібних крихт (breadcrumbs)

Реалізувати з розміткою `BreadcrumbList` (schema.org). Шаблони:

- Сервісна практика: `Головна → Практики → Сімейне право`
- Блог-піллар: `Головна → Блог → Сімейне право`
- Стаття: `Головна → Блог → Сімейне право → Аліменти на дитину`
- Локальна: `Головна → Адвокат у Києві → Сімейний адвокат Київ` (або `→ Лівий берег → Позняки`)
- Кейс: `Головна → Кейси → [Назва кейсу]`

Хлібні крихти = додаткові внутрішні посилання вгору по ієрархії + покращення SERP-сніпета.

## 5. Гайдлайни анкорних текстів

| Тип анкора | Коли використовувати | Приклад | Частка |
|------------|----------------------|---------|--------|
| Точний ключ (exact-match) | помірно, з спиці на комерційну/локальну сторінку | «сімейний адвокат Київ» | ~15% |
| Частковий ключ (partial) | основний робочий тип для контекстних посилань | «оскарження рішення ВЛК» | ~35% |
| Тематичний/брендовий | висхідні посилання на піллар, навігація | «Сімейне право», «практика LEGIUS» | ~25% |
| Природний/реченнєвий | усередині тексту статей | «докладніше про поділ майна подружжя» | ~20% |
| Загальний (generic) | CTA, кнопки | «отримати консультацію», «детальніше» | ~5% |

Правила:
- **Уникати переоптимізації:** не більше 1 exact-match анкора на одну цільову сторінку в межах однієї статті.
- Анкор має відповідати змісту цільової сторінки (релевантність).
- Локальні анкори обов'язково містять гео-маркер («Київ», «Позняки», «лівий берег»).
- Не використовувати один і той самий exact-match анкор з десятків сторінок на одну ціль — варіювати.

**Правила гігієни перелінковки:**
- жодних orphan-сторінок (кожна сторінка має ≥1 вхідне внутрішнє посилання, бажано ≥3);
- глибина кліку від головної ≤ 3 для будь-якої комерційної сторінки;
- усі посилання `dofollow` внутрішні; зовнішні джерела законодавства — `rel="nofollow"` за потреби;
- контроль за каскадом: спиці → піллар → сервісна → конверсія.
